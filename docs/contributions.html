Architecture Decisions

Testing Strategy

Dual-layer testing approach combining fast unit tests with comprehensive end-to-end coverage for maximum confidence with minimal execution time.

Unit (Vitest + happy-dom)
  Pure logic, cache, rate limiting
  57 tests, <3s execution

E2E (Playwright)
  Full page rendering, navigation
  37 tests, real browser

Caching Architecture

Three-layer cache system that reduces database load to under 5% of total requests while keeping response times under 13ms average.

Layer 1: Client (browser)
  localStorage, 15s TTL
  Prevents repeated fetches during navigation

Layer 2: Server (memory)
  In-memory cache, 30s TTL
  95%+ of API requests served from cache

Layer 3: Database (Turso/SQLite)
  Only hit on cache miss (~5% of requests)

Rate Limiting

SQLite-backed persistent rate limiting that survives server restarts and works correctly in serverless environments, with an in-memory fallback.

Storage: SQLite (persistent)
  Survives server restarts
  Works in serverless (Vercel)

Limits:
  GET  /api/predictions: 60 req/min per IP
  POST /api/predictions: 10 req/min per IP

Fallback: In-memory if DB unavailable

Authentication Flow

Twitch OAuth integration via auth-astro with session-based voting and vote deduplication (one vote per user per combat).

Provider: Twitch OAuth (auth-astro + @auth/core)

Flow:
  Login → Twitch OAuth → Callback → Session cookie
  Vote → POST /api/predictions
  Dedup: 1 vote per user per combat
